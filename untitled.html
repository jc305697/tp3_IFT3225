Rapport

Les choix

Nous avons choisi de créer une table de joueur afin de centraliser l'information sur les joueurs.Nous avons mis comme contrainte que le login (username) du joueur est unique à ce dernier pour des raisons de réalisme. Nous avons aussi créer une table des terrains et des heures afin que si une modification au nombre de terrain ou sur les heures d'ouvertures arrivait alors le changement ne nécessiterait pas de très grand effort pour faire les changements dans le système. Puisque mysql n'effectue pas les check et nous n'avions pas les permissions pour faire des triggers nous avons optez pour des clés étrangères. Nous avons donc été limité par mysql et les circonstances. Pour les réservations nous avons créé un table qui utilise la clé primaire de la table joueur.

Apprentissage

Cet exercice a permis une familiarisation avec PHP et mysql.
Il a aussi permis de rappeler l'importance de bien planifier son code avant de commencer son implémentation .
En effet, avoir un plan de conception en tête avec une liste de variables et de méthodes est important et un shéma de baase de données
et permet de sauver du temps en débuggage.

Ce travail pratique a été une excellente occasion de mettre en pratique la partie théorique du cours sur
PHP et mysql.
Ce fût un moment d'apprentissage pour l'utilisation d'une base de données sur un serveur.
